# Memory

## When To Use

Use when the task asks about prior context, settled decisions, project
continuity, or learning capture.

## Operating Rules

- Route through `routing/index.md` before scanning memory broadly.
- Read only the memory files justified by the task.
- Write to `learning/inbox.md` for raw reusable lessons.
- Promote to `memory/` only when the knowledge is durable and categorized.

## Workflow

1. Identify whether the task is lookup, capture, or distillation.
2. Use routing maps to choose relevant files.
3. Read or write the smallest useful context.
4. Preserve why the memory matters for future work.

## Output Expectations

Name the memory files read or changed when that helps future traceability.
