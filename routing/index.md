# Routing Index

Updated: 2026-05-26

Use when: A task is non-trivial, asks about continuity, mentions known projects
or topics, invokes a skill, or may create reusable learning.

Routing is a readable map, not a classifier. Use it to choose the smallest
relevant context set.

## Route Order

1. Read `skill-map.md` when the user asks for a workflow, review, research,
   Algorithm, memory, or audit discipline.
2. Read `projects.md` when a task names a repository, product, client, or
   ongoing workstream.
3. Read `topics.md` when a task names a recurring concept such as memory,
   routing, learning, pi.dev, or Algorithm.
4. Read `aliases.md` when local vocabulary, abbreviations, or names are
   ambiguous.

## Write Rules

Add routes only when they reduce future context gathering. Keep entries short
and point to source-of-truth files rather than duplicating their content.
