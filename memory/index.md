# Memory Index

Updated: 2026-05-26

Use when: A task depends on durable facts, project context, prior decisions,
reusable patterns, people context, or previous sessions.

Memory is intentionally empty in Phase 1. Add durable knowledge through
explicit work or later distillation from `learning/inbox.md`; do not
auto-capture every turn.

## Sections

- `facts/` stores stable facts that are broadly useful.
- `projects/` stores project-specific context and decisions.
- `decisions/` stores durable choices with rationale.
- `patterns/` stores reusable workflows and implementation lessons.
- `people/` stores relationship or preference context when explicitly useful.
- `sessions/` stores concise summaries of meaningful work sessions.

## Use Rules

Read the smallest relevant file set. If no matching memory exists, say so and
proceed from the current task context.
